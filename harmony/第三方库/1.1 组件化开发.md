依赖本地的库
```
1、新建一个moudule，注意选择Ability 要选Share Library
2、查看moudule.json5 中的字段type，确保值是shared。
3、到了关键步骤，Index.ets 中添加你要导出的类。 比如：export {HomePage} from "./src/main/ets/pages/HomePage"
4、在entry 的oh-package.json5 的 dependencies 添加  "home": "file:../home"，这样才算依赖home这个库
5、使用的时候，添加 import {HomePage} from 'home' 

注意：如果你发现HomePage() 一直报错，查看HomePage是否声明成export。
必须是 export struct HomePage

另外：在项目的build-profile.json5 （最外层）
 "modules": [
    {
      "name": "library",
      "srcPath": "./library"
    }
  ]


注意：上面的modules 这个要有，entry才能正常依赖library
```

不能运行问题
```
报错：MSG_ERR_INSTALL_DEPENDENT_MODULE_NOT_EXIST

解决：
1、在运行配置页面 点击 Deploy Multi Hap
2、将 Deploy Multi Hap Packages 勾选上
3、选中新增的module了就行了。

```

HAR 与 HSP 区别
```
har 是静态共享库，能打包发布给其他应用引用，包括三方库
hsp 通常是随应用打包。hsp适合单个项目按功能模块分包。

```
