

git patch 打补丁
```
作用：在不同代码库或分支之间共享代码变更的常用方式。
尤其在不直接联网推送代码的场景下，比如通过邮件发送代码变更。

一个Patch 文件是一个文本文件（.patch 或者 .diff）
描述了两次提交差异内容。通过这个文件将相同的更改 复现到另一个代码库或分支上

生成Patch 文件：
1、生成特定提交的Patch （最常用）
git format-patch HEAD~1    //生成最近一个提交的patch

git format_patch HEAD~2     //生成最近两个提交的patch

2、应用Patch 文件：
git am 0001-first-commit.patch 0002-second-commit.patch   //这样当前分支，就会有两个完全一样的提交。
```
