移植和子系统开发

移植
```
将鸿蒙系统适配到新的硬件平台或芯片架构上，让它能在特定设备上运行

核心工作：
硬件抽象层适配：实现鸿蒙内核，对cpu，内存，外设的驱动支持。
内核移植：针对不同芯片架构修改内核代码，确保任务调度，中断管理等机制正常运行
工具链适配：交叉编译工具链的调整，确保系统镜像能正常编译和烧录
启动流程对接：从Bootloader到内核初始化的完整启动链适配

场景：
将鸿蒙移植到新的IoT设备（如智能家居模组，车载芯片）
支持非ARM架构芯片运行鸿蒙

技术栈：
熟悉硬件手册、设备树、BSP开发。
掌握c语言、汇编、内核调试
```

子系统开发
```
基于鸿蒙现有框架，开发或者扩展系统功能模块，增强上层服务能力
核心工作：
子系统功能实现：如开发新额分布式能力（多设备协同）、安全模块（TEE集成）
图形渲染优化等。
框架层开发：基于鸿蒙HDF（硬件驱动框架）或 Ability框架，提供标准化接口供应用调用。
跨子系统协作：确保新模块与其他子系统（如电源管理、文件系统）兼容

场景：
为鸿蒙新增AI子系统（如NPU加速支持）。
开发定制化的轻量级文件系统（如针对穿戴设备的存储优化）

技术栈：
熟悉鸿蒙分层架构（内核层、系统服务层、框架层）
掌握c++/JavaScript/ArkUI （取决于子系统层级）
```
