
内核
```
内核是操作系统的核心，负责管理系统的底层资源（如CPU、内存、进程、文件系统等），
提供基础服务（如任务调度、内存分配、进程间通信IPC等）

主要职责：
系统资源管理：调度任务、分配内存、控制硬件权限
多内核架构支持：鸿蒙支持Linux 宏内核 和 LiteOS 轻量内核
提供底层接口：为上层驱动和应用程序提供系统调用（System Call）和 API

开发内容：
移植或适配内核到不同硬件平台
配置内核参数（如任务调度策略、内存管理策略）
扩展内核功能（新增IPC机制）
```
驱动
```
驱动是内核和硬件之间的桥梁

主要职责：
硬件抽象：将硬件操作封装成标准接口（open(), read()），供上层应用调用
设备管理：初始化设备、处理中断、管理数据传输
兼容性适配：针对不同硬件型号实现差异化支持

开发内容：
实现HDF驱动的框架代码（Bind、Init）
编写设备树配置 或寄存器操作代码
调式硬件兼容性

```
